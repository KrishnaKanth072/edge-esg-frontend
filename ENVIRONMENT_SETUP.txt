EDGE ESG FRONTEND - ENVIRONMENT SETUP GUIDE
===========================================

Your frontend now has 3 environments matching the backend structure:
1. Development (local)
2. Staging (testing)
3. Production (live for 10 companies)


ENVIRONMENT STRUCTURE
---------------------

Backend Environments:
├── dev/ (1 replica, debug logging)
├── staging/ (2 replicas, info logging)
└── production/ (3-5 replicas, warn logging)

Frontend Environments:
├── .env.development (localhost, demo mode, debug)
├── .env.staging (staging URL, testing, info)
└── .env.production (production URL, live, error only)

SETUP INSTRUCTIONS
------------------

STEP 1: LOCAL DEVELOPMENT
--------------------------
1. Copy development environment:
   cp environments/.env.development .env.local

2. Start backend:
   cd ../edge-esg-backend
   docker-compose up -d

3. Start frontend:
   npm run dev

4. Open: http://localhost:3000


STEP 2: STAGING ENVIRONMENT
----------------------------
1. Create staging branch:
   git checkout -b staging

2. Push to GitHub:
   git push -u origin staging

3. Setup Vercel Staging:
   - Go to Vercel Dashboard
   - Add new project or use existing
   - Connect staging branch
   - Add environment variables from environments/.env.staging
   - Set environment to "Preview"

4. Configure GitHub Secrets for Staging:
   Go to: Settings → Secrets → Actions → New secret
   
   Add these secrets:
   - STAGING_API_URL=https://staging-api.yourdomain.com
   - STAGING_WS_URL=wss://staging-api.yourdomain.com/ws
   - STAGING_KEYCLOAK_URL=https://staging-auth.yourdomain.com
   - STAGING_KEYCLOAK_REALM=edgeesg
   - STAGING_KEYCLOAK_CLIENT_ID=edge-esg-frontend

5. Deploy:
   git push origin staging
   # Workflow automatically deploys to staging


STEP 3: PRODUCTION ENVIRONMENT
-------------------------------
1. Ensure main branch is ready:
   git checkout main
   git merge staging  # After staging testing

2. Setup Vercel Production:
   - Go to Vercel Dashboard
   - Connect main branch
   - Add environment variables from environments/.env.production
   - Set environment to "Production"

3. Configure GitHub Secrets for Production:
   Go to: Settings → Secrets → Actions → New secret
   
   Add these secrets:
   - PRODUCTION_API_URL=https://api.yourdomain.com
   - PRODUCTION_WS_URL=wss://api.yourdomain.com/ws
   - PRODUCTION_KEYCLOAK_URL=https://auth.yourdomain.com
   - PRODUCTION_KEYCLOAK_REALM=edgeesg
   - PRODUCTION_KEYCLOAK_CLIENT_ID=edge-esg-frontend

4. Deploy:
   git push origin main
   # Workflow automatically deploys to production


VERCEL ENVIRONMENT SETUP
-------------------------

Development (Preview Deployments):
1. Go to: Project Settings → Environment Variables
2. Select: "Preview" environment
3. Add variables from environments/.env.development

Staging (Staging Branch):
1. Go to: Project Settings → Environment Variables
2. Select: "Preview" environment
3. Add variables from environments/.env.staging
4. Or create separate Vercel project for staging

Production (Main Branch):
1. Go to: Project Settings → Environment Variables
2. Select: "Production" environment
3. Add variables from environments/.env.production
4. Update URLs with actual production values

ROLLBACK STRATEGY
-----------------

If production deployment fails:

1. Immediate Rollback (Vercel):
   - Go to Vercel Dashboard
   - Deployments tab
   - Find last working deployment
   - Click "Promote to Production"

2. Git Rollback:
   git revert HEAD
   git push origin main
   # Auto-deploys previous version

3. Emergency Fix:
   - Fix on hotfix branch
   - Test in staging
   - Fast-track to production
