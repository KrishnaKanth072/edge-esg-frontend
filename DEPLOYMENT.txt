EDGE ESG FRONTEND - PRODUCTION DEPLOYMENT GUIDE
================================================

STEP 1: PUSH TO GITHUB
-----------------------
1. Create new repository on GitHub: https://github.com/new
   Name: edge-esg-frontend
   Description: AI-powered ESG trading intelligence dashboard
   Public or Private: Your choice
   DO NOT initialize with README (we already have one)

2. Push code to GitHub:
   git remote add origin https://github.com/YOUR_USERNAME/edge-esg-frontend.git
   git branch -M main
   git push -u origin main


STEP 2: DEPLOY TO VERCEL (FREE)
--------------------------------
1. Go to: https://vercel.com/signup
   Sign up with your GitHub account (FREE forever)

2. Import your repository:
   - Click "Add New" → "Project"
   - Select "edge-esg-frontend" repository
   - Click "Import"

3. Configure project:
   Framework Preset: Next.js (auto-detected)
   Root Directory: ./
   Build Command: npm run build (auto-detected)
   Output Directory: .next (auto-detected)

4. Add Environment Variables:
   Click "Environment Variables" and add:
   
   NEXT_PUBLIC_API_URL=http://YOUR_BACKEND_IP:8000
   NEXT_PUBLIC_WS_URL=ws://YOUR_BACKEND_IP:8000/ws
   NEXT_PUBLIC_KEYCLOAK_URL=http://YOUR_BACKEND_IP:8080
   NEXT_PUBLIC_KEYCLOAK_REALM=edgeesg
   NEXT_PUBLIC_KEYCLOAK_CLIENT_ID=edge-esg-frontend

   NOTE: Replace YOUR_BACKEND_IP with your actual backend server IP
   If running backend locally, use your public IP or ngrok

5. Click "Deploy"
   Wait 2-3 minutes for deployment to complete

6. Your app will be live at: https://edge-esg-frontend-YOUR_USERNAME.vercel.app


STEP 3: CONFIGURE BACKEND FOR PRODUCTION
-----------------------------------------
1. Update backend CORS to allow your Vercel URL:
   Set environment variable: FRONTEND_URL=https://edge-esg-frontend-YOUR_USERNAME.vercel.app

2. Update Keycloak redirect URIs:
   - Login to Keycloak: http://YOUR_BACKEND_IP:8080
   - Go to: Clients → edge-esg-frontend → Settings
   - Add to "Valid Redirect URIs": https://edge-esg-frontend-YOUR_USERNAME.vercel.app/*
   - Add to "Web Origins": https://edge-esg-frontend-YOUR_USERNAME.vercel.app
   - Click "Save"


STEP 4: TEST PRODUCTION DEPLOYMENT
-----------------------------------
1. Open your Vercel URL in browser
2. Check backend connection status (top of page)
3. Enter company name: "Tata Steel"
4. Click "Analyze ESG + Trading Alpha"
5. Watch 10 agents animate
6. View results in 4 glass cards

If backend is offline, app automatically uses demo mode.


STEP 5: CUSTOM DOMAIN (OPTIONAL)
---------------------------------
1. In Vercel dashboard, go to: Settings → Domains
2. Add your custom domain: edge-esg.yourdomain.com
3. Update DNS records as instructed by Vercel
4. SSL certificate is automatically provisioned (FREE)


ALTERNATIVE: RUN BACKEND LOCALLY WITH NGROK
--------------------------------------------
If you don't have a public server for backend:

1. Install ngrok: https://ngrok.com/download (FREE)
2. Start backend: docker-compose up -d
3. Expose backend: ngrok http 8000
4. Copy ngrok URL: https://abc123.ngrok.io
5. Update Vercel environment variables:
   NEXT_PUBLIC_API_URL=https://abc123.ngrok.io
   NEXT_PUBLIC_WS_URL=wss://abc123.ngrok.io/ws
6. Redeploy on Vercel
