PUSH TO GITHUB & DEPLOY - STEP BY STEP
=======================================

STEP 1: CREATE GITHUB REPOSITORY
---------------------------------
1. Go to: https://github.com/new
2. Repository name: edge-esg-frontend
3. Description: AI-powered ESG trading intelligence dashboard
4. Public or Private: Your choice
5. DO NOT initialize with README (we already have one)
6. Click "Create repository"


STEP 2: PUSH FRONTEND TO GITHUB
--------------------------------
Copy and paste these commands ONE BY ONE:

cd edge-esg-frontend

git remote add origin https://github.com/YOUR_USERNAME/edge-esg-frontend.git

git branch -M main

git push -u origin main


Replace YOUR_USERNAME with your GitHub username (krishnakanth072)


STEP 3: VERIFY ON GITHUB
-------------------------
1. Go to: https://github.com/YOUR_USERNAME/edge-esg-frontend
2. You should see all your files
3. Check that workflows are in .github/workflows/


STEP 4: DEPLOY TO VERCEL (FREE)
--------------------------------
1. Go to: https://vercel.com/signup
2. Click "Continue with GitHub"
3. Authorize Vercel to access your GitHub

4. Click "Add New" → "Project"
5. Find "edge-esg-frontend" repository
6. Click "Import"

7. Configure Project:
   Framework Preset: Next.js (auto-detected)
   Root Directory: ./
   Build Command: npm run build
   Output Directory: .next
   Install Command: npm install

8. Add Environment Variables:
   Click "Environment Variables"
   
   Add these (for now, use localhost - we'll update later):
   
   Name: NEXT_PUBLIC_API_URL
   Value: http://localhost:8000
   
   Name: NEXT_PUBLIC_WS_URL
   Value: ws://localhost:8000/ws
   
   Name: NEXT_PUBLIC_KEYCLOAK_URL
   Value: http://localhost:8080
   
   Name: NEXT_PUBLIC_KEYCLOAK_REALM
   Value: edgeesg
   
   Name: NEXT_PUBLIC_KEYCLOAK_CLIENT_ID
   Value: edge-esg-frontend

9. Click "Deploy"

10. Wait 2-3 minutes

11. Your app will be live at:
    https://edge-esg-frontend-YOUR_USERNAME.vercel.app


STEP 5: ENABLE DEPENDABOT
--------------------------
1. Go to: https://github.com/YOUR_USERNAME/edge-esg-frontend
2. Click "Settings" tab
3. Click "Security" → "Code security and analysis"
4. Enable "Dependabot alerts"
5. Enable "Dependabot security updates"


STEP 6: TEST FRONTEND DEPLOYMENT
---------------------------------
1. Open your Vercel URL in browser
2. You should see the EDGE ESG dashboard
3. Backend status will show "Demo Mode (Backend Offline)"
4. This is NORMAL - backend is still on localhost
5. Try entering "Tata Steel" and click Analyze
6. It will use demo/simulation mode


STEP 7: DEPLOY BACKEND (CHOOSE ONE OPTION)
-------------------------------------------

OPTION A: Use ngrok (Quick Testing)
------------------------------------
1. Download ngrok: https://ngrok.com/download
2. Extract and run:
   
   cd edge-esg-backend
   docker-compose up -d
   
   # In another terminal
   ngrok http 8000

3. Copy the HTTPS URL (e.g., https://abc123.ngrok.io)

4. Update Vercel Environment Variables:
   - Go to Vercel Dashboard → Your Project → Settings → Environment Variables
   - Update NEXT_PUBLIC_API_URL to: https://abc123.ngrok.io
   - Update NEXT_PUBLIC_WS_URL to: wss://abc123.ngrok.io/ws
   - Click "Save"

5. Redeploy:
   - Go to Deployments tab
   - Click "..." on latest deployment
   - Click "Redeploy"

⚠️ Note: ngrok URL changes when you restart


OPTION B: Deploy Backend to Railway (FREE)
-------------------------------------------
1. Go to: https://railway.app
2. Sign up with GitHub
3. Click "New Project"
4. Select "Deploy from GitHub repo"
5. Choose "edge-esg-backend"
6. Railway will detect Docker Compose
7. Configure services (PostgreSQL, Redis, etc.)
8. Get your Railway URL
9. Update Vercel environment variables with Railway URL


OPTION C: Keep Backend Local (Testing Only)
--------------------------------------------
1. Run backend locally:
   cd edge-esg-backend
   docker-compose up -d

2. Run frontend locally:
   cd edge-esg-frontend
   npm run dev

3. Open: http://localhost:3000
4. Both will work together on localhost


STEP 8: UPDATE BACKEND CORS
----------------------------
Once you have your Vercel URL:

1. Open: edge-esg-backend/internal/middleware/cors.go
2. It's already configured to accept Vercel URLs
3. Set environment variable:
   export FRONTEND_URL=https://edge-esg-frontend-YOUR_USERNAME.vercel.app

4. Restart backend:
   docker-compose down
   docker-compose up -d


STEP 9: UPDATE KEYCLOAK
------------------------
1. Login to Keycloak: http://localhost:8080 (admin/admin)
2. Go to: Clients → edge-esg-frontend → Settings
3. Add to "Valid Redirect URIs":
   https://edge-esg-frontend-YOUR_USERNAME.vercel.app/*
4. Add to "Web Origins":
   https://edge-esg-frontend-YOUR_USERNAME.vercel.app
5. Click "Save"


STEP 10: TEST PRODUCTION
-------------------------
1. Open your Vercel URL
2. Check backend connection status (should be "Connected" if backend is public)
3. Enter "Tata Steel"
4. Click "Analyze ESG + Trading Alpha"
5. Watch 10 agents animate
6. View results


TROUBLESHOOTING
---------------

Frontend not deploying?
- Check Vercel deployment logs
- Verify all environment variables are set
- Check build logs for errors

Backend not connecting?
- Verify backend is running: curl http://localhost:8000/health
- Check CORS configuration
- Verify environment variables in Vercel

Can't push to GitHub?
- Check if remote is set: git remote -v
- Verify GitHub credentials
- Try: git push -u origin main --force (if needed)
