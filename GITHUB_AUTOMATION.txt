EDGE ESG FRONTEND - GITHUB AUTOMATION GUIDE
===========================================

All GitHub workflows and automation rules from backend are now applied to frontend!


AUTOMATED WORKFLOWS
-------------------

1. AUTO-FIX VULNERABILITIES (.github/workflows/auto-fix-vulnerabilities.yml)
   ✅ Runs: Every Monday at 9 AM UTC + Manual trigger + On package.json changes
   ✅ What it does:
      - Scans for security vulnerabilities with npm audit
      - Automatically fixes vulnerabilities with npm audit fix
      - Tests build to ensure nothing broke
      - Creates PR with fixes for your review
      - Assigns to @krishnakanth072
   ✅ Auto-merge: Safe patch updates are auto-merged after approval

2. AUTO-MERGE DEPENDABOT (.github/workflows/auto-merge-dependabot.yml)
   ✅ Runs: On every Dependabot PR
   ✅ What it does:
      - Security updates: Auto-merge immediately
      - Patch updates (x.y.Z): Auto-merge
      - Minor updates (x.Y.z): Auto-merge
      - Major updates (X.y.z): Requires manual review
   ✅ Saves time: No manual approval needed for safe updates

3. REQUIRE ADMIN APPROVAL (.github/workflows/require-approval.yml)
   ✅ Runs: On every PR
   ✅ What it does:
      - PRs from @krishnakanth072: Auto-approved
      - PRs from others: Requires @krishnakanth072 approval
      - Blocks merge until approved
   ✅ Security: Only admin can approve changes

4. PR CHECKS (.github/workflows/pr-checks.yml)
   ✅ Runs: On every PR to main/master
   ✅ What it does:
      - Lint: Runs ESLint
      - Type Check: Validates TypeScript
      - Build: Tests production build
      - Security: Scans for hardcoded secrets
      - Bundle Size: Checks build size
   ✅ Quality: Ensures code quality before merge

5. SECURITY SCAN (.github/workflows/security-scan.yml)
   ✅ Runs: Daily at 2 AM UTC + On push to main/master
   ✅ What it does:
      - NPM Audit: Checks for vulnerabilities
      - Dependency Check: Finds outdated packages
      - Code Scan: Looks for sensitive data
      - Trivy Scan: Filesystem vulnerability scan
   ✅ Protection: Continuous security monitoring

6. PRODUCTION DEPLOY (.github/workflows/deploy.yml)
   ✅ Runs: On push to main/master + Manual trigger
   ✅ What it does:
      - Installs dependencies
      - Runs linter
      - Type checks TypeScript
      - Security audit
      - Builds for production
      - Shows deployment summary
   ✅ Ready: Vercel auto-deploys from this

7. DEPENDENCY CHECK (.github/workflows/dependency-check.yml)
   ✅ Runs: Daily at midnight
   ✅ What it does:
      - Runs npm audit
      - Checks for outdated packages
   ✅ Awareness: Stay informed about updates

HOW IT WORKS
------------

SCENARIO 1: Security Vulnerability Found
   1. Dependabot detects vulnerability
   2. Creates PR with fix
   3. Auto-merge workflow approves it
   4. PR is merged automatically
   5. Deploy workflow builds and deploys
   6. You get notification

SCENARIO 2: Weekly Vulnerability Scan
   1. Monday 9 AM: Auto-fix workflow runs
   2. Scans for vulnerabilities
   3. Applies fixes automatically
   4. Creates PR for review
   5. Assigns to you
   6. You review and merge

SCENARIO 3: Someone Creates PR
   1. Developer creates PR
   2. PR checks run (lint, type, build, security)
   3. Require-approval workflow blocks merge
   4. Waits for @krishnakanth072 approval
   5. After approval, PR can be merged

SCENARIO 4: You Create PR
   1. You create PR
   2. PR checks run
   3. Require-approval auto-approves (you're admin)
   4. You can merge immediately

TROUBLESHOOTING
---------------

Workflow failed?
   - Check Actions tab for error logs
   - Click on failed workflow
   - Review error messages
   - Fix and push again

Dependabot not creating PRs?
   - Check Settings → Security → Dependabot
   - Ensure it's enabled
   - Check .github/dependabot.yml syntax

Auto-merge not working?
   - Check PR labels
   - Verify Dependabot created the PR
   - Check workflow permissions

Need help?
   - Check workflow logs in Actions tab
   - Review this guide
   - Check GitHub documentation
